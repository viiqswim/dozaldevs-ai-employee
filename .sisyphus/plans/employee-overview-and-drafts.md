# Employee Creation: Structured Overview + Draft Persistence

## TL;DR

> **Quick Summary**: Replace the raw AI-prompt preview with a human-readable structured overview, and add draft persistence so generated employee configs survive page refresh/close with their own URL.
>
> **Deliverables**:

> - Structured 6-section employee overview (Role, Trigger, Workflow, Tools Used, Output, Approval) generated by Haiku as JSON

> - New review/edit page at `/dashboard/employees/:archetypeId/edit`

> - Production safety filters: dispatch/poll/classifier queries filter `status=active` only

## Context

### Original Request

Two features for the AI employee creation flow:

1. Replace the flat "What it does" with a structured, human-readable overview — keeping raw AI fields in a collapsed Advanced section with honest labels
2. Add draft persistence so generated configs survive page refresh/close, with their own URL and draft badges in the employees list

### Interview Summary

**Key Decisions**:

- Structured overview: 6 sections generated as JSON — Role, Trigger, Workflow, Tools Used, Output, Approval
- Advanced section relabels: "Trigger Prompt", "Employee Brain", "Delivery Instructions"
- Auto-save: generation finishes → draft saved to DB immediately → redirect to review page
- Refine: each refinement creates a NEW draft record; previous draft gets `status: "superseded"`
- Finalize: PATCH existing draft → `status: "active"`. Same ID persists.
- Multiple drafts per tenant, no expiration, no deletion UI in this iteration
- Drafts appear in employees list with "Draft" badge
- Tests after implementation

**Research Findings**:

- React Router v6, flat routes. Creation at `/dashboard/employees/new`
- EmployeeList fetches via PostgREST with no filtering — shows everything
- `@@unique([tenant_id, role_name])` constraint exists — blocks draft + active with same name
- Schema pattern: `status String @default("active")` (not enums)
- `create-task-and-dispatch.ts` lookups have no status filter — production safety risk
- PATCH endpoint missing: `agents_md`, `delivery_instructions`, `trigger_sources`, `tool_registry`

### Metis Review (Key Findings Applied)

- `role_name` uniqueness: resolved with partial unique index (`WHERE status = 'active'`)
- Refine orphan accumulation: `status: "superseded"` for replaced drafts
- `notification_channel` not in GenerateArchetypeResponse: explicitly included in draft POST
- Field naming: using `overview` not `description` (avoids collision with user input)
- `maxTokens: 4000 → 6000` for overview generation
- Safety filters added on dispatch/poll/classifier as mandatory tasks
