# Guest Messaging — 01: Approve & Send (Happy Path)

> **Purpose**: Verify the end-to-end happy path — a guest message arrives, the AI drafts a response, a Slack card appears with approval buttons, you approve it, and the message is delivered to the guest via Hostfully.

> **Prerequisites**: Complete `00-prerequisites-and-setup.md` before running this test.

## Step 1 — Send the Webhook

Use a unique `message_uid` each time you run this test (change the suffix).

```bash
source .env

curl -s -X POST http://localhost:7700/webhooks/hostfully \
  -H "Content-Type: application/json" \
  -d '{
    "agency_uid": "942d08d9-82bb-4fd3-9091-ca0c6b50b578",
    "event_type": "NEW_INBOX_MESSAGE",
    "message_uid": "test-approve-001",
    "thread_uid": "2f18249a-9523-4acd-a512-20ff06d5c3fa",
    "lead_uid": "37f5f58f-d308-42bf-8ed3-f0c2d70f16fb",
    "property_uid": "c960c8d2-9a51-49d8-bb48-355a7bfbe7e2",
    "message": "Hi, is the property available next weekend? We are a family of 4."
  }' | jq .
```

**Expected**:

```json
{ "ok": true, "task_id": "<uuid>" }
```

Note the `task_id` — you'll use it to check status.

```bash
TASK_ID="<paste task_id here>"
```

## Step 4 — Verify Slack Card Appears

Open the VLRE Slack workspace and navigate to the channel `C0960S2Q8RL`.

**Expected Slack card structure**:

- Header block with the task summary (guest name, property, message snippet)
- The AI's draft response text
- Three action buttons:
  - `✅ Approve & Send`
  - `✏️ Edit & Send`
  - `❌ Reject`
- A trailing context block showing `Task \`<task_id>\``

---

## Step 5 — Click "Approve & Send"

Click the `✅ Approve & Send` button in Slack.

**Expected immediate response** (Slack card updates in-place):

```
⏳ Processing approval...
```

Then within a few seconds, the card updates to:

```
✅ Approved by @<your-name> — delivering now.
```
