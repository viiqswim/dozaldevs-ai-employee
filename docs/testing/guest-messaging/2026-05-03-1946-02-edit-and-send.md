# Guest Messaging — 02: Edit & Send

> **Purpose**: Verify the edit path — the AI drafts a response, you open the edit modal, modify the text, and the edited version (not the original) is delivered to the guest.

> **Prerequisites**: Complete `00-prerequisites-and-setup.md` before running this test.

---

| Gateway                | `http://localhost:7700`                |

## Step 1 — Send the Webhook

Use a unique `message_uid` each time (change the suffix).

```bash
source .env

curl -s -X POST http://localhost:7700/webhooks/hostfully \
  -H "Content-Type: application/json" \
  -d '{
    "agency_uid": "942d08d9-82bb-4fd3-9091-ca0c6b50b578",
    "event_type": "NEW_INBOX_MESSAGE",
    "message_uid": "test-edit-001",
    "thread_uid": "2f18249a-9523-4acd-a512-20ff06d5c3fa",
    "lead_uid": "37f5f58f-d308-42bf-8ed3-f0c2d70f16fb",
    "property_uid": "c960c8d2-9a51-49d8-bb48-355a7bfbe7e2",
    "message": "Hello! What is the check-in time and do you allow pets?"
  }' | jq .
```

**Expected**:

```json
{ "ok": true, "task_id": "<uuid>" }
```

```bash
TASK_ID="<paste task_id here>"
```

---

## Step 2 — Wait for the Slack Card

Monitor the Inngest dashboard at `http://localhost:8288` until the run reaches `wait-for-approval` (status = `Reviewing`).

Verify via API:

```bash
source .env

curl -s "http://localhost:7700/admin/tenants/00000000-0000-0000-0000-000000000003/tasks/$TASK_ID" \
  -H "Authorization: Bearer $SERVICE_TOKEN" | jq '{status, deliverable}'
```

**Expected**: `"status": "Reviewing"` with a `draftResponse` in the deliverable.

Note the original `draftResponse` text — you'll verify the edited version replaces it.

---

## Step 3 — Note the Original Draft

From the API response above, copy the `draftResponse` value. This is what the AI generated. You will edit it in the next step.

Also check the Slack card in `C0960S2Q8RL` — it should show the same draft text and three buttons:

- `✅ Approve & Send`
- `✏️ Edit & Send` ← you will click this
- `❌ Reject`

---

## Step 4 — Click "Edit & Send"

Click the `✏️ Edit & Send` button in Slack.

**Expected**: A modal dialog opens with:

- Title: something like "Edit Response"
- A text area pre-filled with the AI's `draftResponse`
- A "Send Edited Response" (or "Submit") button

---

## Step 5 — Edit the Text and Submit

1. Clear the pre-filled text or modify it
2. Type your edited response, e.g.:
   ```
   Hi! Check-in is at 3pm. We do allow small pets (under 25 lbs) with a $50 pet fee. Let us know if you have any other questions!
   ```
3. Click "Send Edited Response" (or the modal's submit button)

**Expected immediate response** (modal closes, Slack card updates):

```
⏳ Processing approval...
```

Then within a few seconds:

```
✅ Approved by @<your-name> — delivering now.
```

---

## Step 6 — Verify the Deliverable Was Updated

```bash
curl -s "http://localhost:7700/admin/tenants/00000000-0000-0000-0000-000000000003/tasks/$TASK_ID" \
  -H "Authorization: Bearer $SERVICE_TOKEN" | jq '{status, deliverable}'
```

**Expected**:

- `"status": "Approved"` (briefly) or `"Done"` (if delivery already completed)
- `deliverable.draftResponse` should contain **your edited text**, not the original AI draft

> This confirms the lifecycle correctly patches the deliverable with `editedContent` before delivery.

## ✅ Test Passed

The edited text was delivered — not the original AI draft. The edit modal pre-fill, submission, deliverable patch, and delivery all worked correctly.

**If the original draft was sent instead of your edit**, check:

- The `patch-deliverable` step in Inngest — did it run?
- The `deliverable.draftResponse` in the DB after approval — was it updated?
- The delivery machine logs: `fly logs -a ai-employee-workers`
