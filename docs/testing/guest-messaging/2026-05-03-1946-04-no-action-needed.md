## Background: What Triggers NO_ACTION_NEEDED?

The AI classifies a message as `NO_ACTION_NEEDED` when it determines the guest's message does not require a response — for example:

- A "thank you" message after check-out
- A confirmation that the guest received information
- A message that was already answered in a prior turn

The exact classification depends on the AI's judgment based on the message content and conversation history.

## Sub-Path A: Let It Time Out (Task → Done)

> **Note**: The actual timeout is 24 hours. For testing purposes, you can verify the waiting state and trust the timeout behavior, or use the Inngest dashboard to manually advance the sleep step.

**To verify the waiting state**:

```bash
source .env

curl -s "http://localhost:7700/admin/tenants/00000000-0000-0000-0000-000000000003/tasks/$TASK_ID" \
  -H "X-Admin-Key: $ADMIN_API_KEY" | jq '{status}'
```

**Expected while waiting**: `"status": "Reviewing"` (the task stays in Reviewing while waiting for the reply-anyway event).

**After 24h timeout** (or after manually advancing in Inngest):

- Task status → `Done`
- No message sent to guest

To manually trigger the timeout in Inngest (for testing without waiting 24h):

1. Open the run in the Inngest dashboard
2. Find the `wait-for-reply-anyway` sleep step
3. Use the "Cancel sleep" or "Skip" option if available in your Inngest version

1. Click the `💬 Reply Anyway` button in Slack
