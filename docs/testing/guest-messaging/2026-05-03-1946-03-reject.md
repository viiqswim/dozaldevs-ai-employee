# Guest Messaging — 03: Reject

> **Purpose**: Verify the rejection path — the AI drafts a response, you reject it (with or without a reason), the task is cancelled, and NO message is sent to the guest.

> **Prerequisites**: Complete `00-prerequisites-and-setup.md` before running this test.

## Step 1 — Send the Webhook

Use a unique `message_uid` each time (change the suffix).

```bash
source .env

curl -s -X POST http://localhost:7700/webhooks/hostfully \
  -H "Content-Type: application/json" \
  -d '{
    "agency_uid": "942d08d9-82bb-4fd3-9091-ca0c6b50b578",
    "event_type": "NEW_INBOX_MESSAGE",
    "message_uid": "test-reject-001",
    "thread_uid": "2f18249a-9523-4acd-a512-20ff06d5c3fa",
    "lead_uid": "37f5f58f-d308-42bf-8ed3-f0c2d70f16fb",
    "property_uid": "c960c8d2-9a51-49d8-bb48-355a7bfbe7e2",
    "message": "Can I get a discount if I book for 2 weeks?"
  }' | jq .
```

**Expected**:

```json
{ "ok": true, "task_id": "<uuid>" }
```

```bash
TASK_ID="<paste task_id here>"
```

## Step 2 — Wait for the Slack Card

Monitor the Inngest dashboard at `http://localhost:8288` until the run reaches `wait-for-approval`.

Verify:

```bash
source .env

curl -s "http://localhost:7700/admin/tenants/00000000-0000-0000-0000-000000000003/tasks/$TASK_ID" \
  -H "X-Admin-Key: $ADMIN_API_KEY" | jq '{status}'
```

**Expected**: `"status": "Reviewing"`

The Slack card in `C0960S2Q8RL` should show three buttons:

- `✅ Approve & Send`
- `✏️ Edit & Send`
- `❌ Reject` ← you will click this

---

## Step 3 — Click "Reject"

Click the `❌ Reject` button in Slack.

**Expected**: A modal dialog opens with:

- Title: something like "Reject Response"
- An optional text field for a rejection reason
- A "Submit" (or "Confirm Rejection") button

---

## Sub-Path A: Reject Without a Reason

1. Leave the reason field empty
2. Click "Submit"

**Expected**: Modal closes, Slack card updates to:

```
❌ Rejected by @<your-name>
```

---

## Sub-Path B: Reject With a Reason

1. Type a reason in the text field, e.g.:
   ```
   The discount policy needs to be checked with the owner first. Do not send.
   ```
2. Click "Submit"

**Expected**: Modal closes, Slack card updates to:

```
❌ Rejected by @<your-name>
Reason: The discount policy needs to be checked with the owner first. Do not send.
```

---

## Step 4 — Verify Task is Cancelled

```bash
curl -s "http://localhost:7700/admin/tenants/00000000-0000-0000-0000-000000000003/tasks/$TASK_ID" \
  -H "X-Admin-Key: $ADMIN_API_KEY" | jq '{id, status, rejection_reason}'
```

**Expected**:

```json
{
  "id": "<task_id>",
  "status": "Cancelled",
  "rejection_reason": "<your reason, or null if Sub-Path A>"
}
```

---

## Step 5 — Verify Inngest Run Ended

In the Inngest dashboard, the run should show:

| Step                | Expected State             |
| ------------------- | -------------------------- |
| `wait-for-approval` | ✅ (received reject event) |
| `mark-cancelled`    | ✅                         |

The run should be in a terminal state (completed/cancelled) — no delivery steps should have run.

---

## Step 6 — Verify NO Message Was Sent to Hostfully

1. Log into the Hostfully platform
2. Navigate to the inbox thread: `https://platform.hostfully.com/app/#/inbox?threadUid=2f18249a-9523-4acd-a512-20ff06d5c3fa&leadUid=37f5f58f-d308-42bf-8ed3-f0c2d70f16fb`
3. Confirm **no new message** was sent from the AI employee after the rejection

> **Critical**: If a message appears in Hostfully after rejection, the delivery gate is broken. File a bug immediately.

---

## ✅ Test Passed

Task is `Cancelled`, rejection reason stored (if provided), and no message was delivered to the guest.

**If a message was sent despite rejection**, check:

- The Inngest run — did the `deliver` step run after `mark-cancelled`?
- The `wait-for-approval` step — did it receive a `reject` event or an `approve` event?
- Gateway logs for any duplicate event processing
